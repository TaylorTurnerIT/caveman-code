# Contributing

Contributions to Caveman are welcome. The process is straightforward—focus on improving the SKILL.md prompt.

## How to Contribute

1. Fork the repository
2. Edit only `skills/caveman/SKILL.md` — this is the single source of truth
3. Submit a pull request that includes:
   - Before/after examples demonstrating the improvement
   - A brief rationale for the change

## Important Notes

**Single source of truth:** Edit `skills/caveman/SKILL.md` — this is the only copy you need to touch. Other copies across the codebase are automatically synchronized after merge, so direct edits to those files will be overwritten.

**Philosophy:** "Small focused change > big rewrite. Caveman like simple."

## Getting Started

Contributors can browse issues labeled "good first issue" for beginner-friendly tasks. The contribution workflow requires providing context showing the current behavior, the proposed improvement, and a concise explanation of why the change is beneficial. This structured approach helps maintain consistency and clarity in the prompt development process.
