# Security Policy

This project maintains a responsible vulnerability disclosure process:

## Reporting Process

Do not create public issues. Instead, submit vulnerability details to security@juliusbr.com, including description, reproduction steps, and supporting materials. Expect initial contact "within 72 hours."

## Version Support

Only the latest version receives security support.

## Disclosure Approach

The project uses coordinated disclosure, meaning vulnerabilities are fixed before public announcement. Security advisories will be released once patches are ready, with reporter attribution offered (optional anonymity respected).

The security team commits to working collaboratively with reporters to address issues before they become public knowledge.
